# PRD (요구사항 문서) — 식문화뱅크 홈페이지

이 문서는 개발자가 “그냥 만들면” 내용이 빠지거나 모양이 달라질 수 있어서,  
**무엇을 어떻게 만들지**를 미리 적어두는 약속 종이예요.

---

## 0) 한 줄 목표
방문자가 홈페이지를 보고 **식문화뱅크가 어떤 곳인지 30초 안에 이해**하고, **바로 연락**할 수 있게 한다.

## 1) 대상(누가 보나요?)
- 식품/영양 교육, 식생활 안전 교육, 컨설팅이 필요한 개인/기관/학교/기업
- 상담을 원하는 사람(전화/이메일)

## 2) 페이지 형태
- **여러 페이지(Multi page)**: 메뉴 클릭 시 페이지가 바뀌는 형태

### 2-1. 페이지(초안)
- `index.html` : 인사말(첫 화면)
- `about.html` : 식문화뱅크 소개 + 비전/미션
- `services.html` : 주요 사업 영역
- `contact.html` : 신청 및 상담(연락처/주소/지도/문의사항)

## 3) 디자인 톤(분위기)
### 3-1. 전체 분위기
- **깨끗함 / 건강함 / 따뜻함**
- 파스텔 계열 그라데이션(파랑~핑크) 같은 부드러운 느낌

### 3-2. 참고 이미지
- `김미옥 사진-.png`
  - 배경: 파스텔 그라데이션 + 하트 워터마크(연하게)
  - 오른쪽에 인물, 왼쪽이 비어있어 글자를 올리기 좋음

### 3-3. 폰트(초안)
- 시스템 기본 폰트(윈도우 기준)로 시작: `Pretendard`가 있으면 사용, 없으면 기본 sans-serif
- 큰 제목은 두껍게, 본문은 읽기 편하게

## 4) 정보 구조(섹션 구성)
각 페이지 안에서 섹션 id는 “페이지 내 이동”에 사용합니다.

- `index.html`
  - `#home` 인사말(첫 화면/히어로)
- `about.html`
  - `#about` 소개/비전/미션
- `services.html`
  - `#services` 주요 사업 영역
- `contact.html`
  - `#contact` 신청 및 상담(연락처/주소/지도/문의사항)

## 5) 섹션별 상세 요구사항(콘텐츠)
### 5-1. `#home` 인사말(첫 화면/히어로)
**목적**: “여기가 무엇을 하는 곳인지” 첫눈에 알게 하기

- **표시 텍스트(원문 기반)**:
  - 제목: `안녕하세요. 식문화뱅크입니다.`
  - 본문(줄바꿈 허용):
    - `저희는 식품과 영양을 바탕으로 한 건강한 삶을 만들어가기 위해 설립된 교육·컨설팅 전문 기업입니다.`
    - `식문화뱅크는 단순한 정보 전달을 넘어, 삶의 현장에서 실천할 수 있는 식생활 지혜를 나누고자 합니다.`
  - (선택) 대표 서명:
    - `식문화뱅크 대표 김미옥`

- **강조 문장/3가지 약속(원문 기반, 카드/목록 형태)**:
  - 첫째: 올바른 식습관 형성/균형 잡힌 영양 지도
  - 둘째: 위생/안전 기준 확산(식생활 안전 교육)
  - 셋째: 맞춤형 식품 및 관련 상품 제공

- **버튼(권장)**:
  - 버튼 텍스트: `상담 신청하기`
  - 동작: 클릭하면 `#contact` 섹션으로 부드럽게 스크롤 이동

- **배경**:
  - 1안: `김미옥 사진-.png`를 히어로 배경 이미지로 사용
  - 2안: 이미지 대신 그라데이션 배경 + 하트 워터마크(옵션)

### 5-2. `#about` 식문화뱅크 소개 / 비전 / 미션
**목적**: 회사 정체성을 “짧고 명확하게” 전달

- 소개(원문 기반):
  - 제목: `식문화뱅크 소개`
  - 본문:
    - `식문화뱅크는 식품·영양 교육과 식생활 안전을 중심으로 한 전문 컨설팅 기업입니다.`
    - `우리는 단순한 교육을 넘어, 현장에서 실천 가능한 멘토링과 콘텐츠 제공을 통해 개인과 사회가 함께 성장할 수 있는 기반을 마련합니다.`

- 비전(원문 기반):
  - 제목: `비전`
  - 인용문 스타일:
    - `“건강한 식문화, 미래의 희망을 키우다”`
  - 설명:
    - `식문화뱅크는 모두가 안심하고 즐길 수 있는 먹거리를 통해 지속 가능한 행복을 추구합니다. 고객과 함께 성장하며 세대를 잇는 행복한 식문화를 만듭니다.`

- 미션(원문 기반):
  - 제목: `미션`
  - 인용문 스타일:
    - `"교육·실천을 잇는 멘토링 허브, 건강·즐거운 식문화로 삶을 풍요롭게"`
  - 목록 3개:
    - 교육과 실천을 연결하는 멘토링 허브
    - 안전과 즐거움이 공존하는 식생활 문화 확산
    - 사회 전반의 건강 수준을 높이는 종합 식문화 플랫폼

### 5-3. `#services` 주요 사업 영역
**목적**: 제공 서비스를 한눈에 보기

- 레이아웃: 2열 또는 4카드(화면 크기에 따라 반응형)
- 카드 구성: 제목 + 한 줄 부제(있으면) + bullet 2개

**카드 1**: 식품·영양 교육 멘토  
- 부제: `전문가의 영양코칭`
- bullet:
  - 전 세대를 대상 맞춤형 영양 교육
  - 식습관 개선 및 균형 잡힌 식생활 지도

**카드 2**: 식생활 안전 교육  
- 부제: `안심 식탁 가이드`
- bullet:
  - 식중독 예방, 위생 관리, 안전 먹거리 교육
  - 기관, 기업, 학교 대상 실습형 안전 프로그램

**카드 3**: 영양상품 판매  
- 부제: `지혜로운 건강 마켓`
- 주의: PDF 텍스트에는 카드 3의 bullet이 카드 2와 동일하게 들어가 있는데, 이 부분은 실제 의도인지 확인 필요(질문 항목에 포함).

**카드 4**: 전문가 네트워크 컨설팅  
- 부제: `푸드 솔루션 파트너`
- bullet:
  - 영양사, 식품 전문가, 안전관리 전문가 협업
  - 기관·기업·학교를 위한 맞춤형 컨설팅 서비스

## 6) 기능 요구사항(JS)
- 메뉴 클릭 시 해당 페이지로 이동
- 페이지 내 앵커(예: `#contact`)는 부드럽게 이동(가능하면)
- “상담 신청하기” 버튼 → `contact.html`로 이동(또는 `contact.html#contact`)
- (문의 폼 사용 시) 필수 입력(이름/연락처/내용) 비어 있으면 안내 메시지

## 7) 반응형(모바일) 요구사항
- 모바일에서는 메뉴를 “햄버거(≡)”로 바꾸거나, 메뉴를 단순화(선택)
- 히어로 영역:
  - 글자가 사진 위에서 안 보이면 배경에 반투명 오버레이 적용
  - 텍스트는 중앙 정렬 또는 좌측 정렬 유지(선택)
- 서비스 카드는 1열로 내려가게

## 8) 접근성/기본 품질
- 글자 대비(너무 연한 글씨 금지)
- 이미지에는 대체 텍스트(alt)
- 전화/이메일은 클릭 가능한 링크(`tel:`, `mailto:`)

## 10) 확인이 필요한 질문(모호한 부분)
아래 답만 주시면, 그걸로 “확정된 PRD”로 만들고 바로 페이지 제작 들어갈 수 있어요.

1) **페이지 형태(확정됨)**
- 여러 페이지(메뉴별 페이지)로 진행합니다.

2) **상단 메뉴 구성(확정됨)**
- 메뉴 이름: `홈 / 인사말 / 주요사업 / 신청·상담`
  - 구현(초안): 홈(인사말)=`index.html`, 인사말=동일 페이지 버튼/링크, 주요사업=`services.html`, 신청·상담=`contact.html`
  - 참고: “인사말”을 별도 페이지로 분리하고 싶으면 알려주세요(현재는 `index.html`에 포함하는 걸 기본으로 잡았습니다).

3) **로고(확정: 로고 파일 있음)**
- 로고 파일을 주시면 상단(헤더) 왼쪽에 배치합니다.
  - 권장 형식: `.svg` 또는 투명 배경 `.png`

4) **영양상품 판매(카드 3) 내용**
- 카드 3 설명 문구가 카드 2와 동일하게 들어가 있어요.
  - “영양상품 판매”에 들어갈 **정확한 bullet 2개**를 알려주실 수 있나요?

5) **지도(확정: 네이버지도)**
- `contact.html`에 네이버지도를 넣습니다.

6) **문의폼(확정: 실제 전송 필요)**
- “진짜로 제출되는 문의폼”을 구현합니다.
- 다만 HTML/CSS/JS만으로는 “메일 발송/DB 저장”이 불가능해서, 아래 중 하나를 선택해야 합니다.
  - A안: **구글폼**으로 연결(가장 쉬움/무료, 디자인은 구글폼 스타일)
  - B안: **외부 폼 서비스(Formspree 등)** 사용(디자인 유지 가능, 무료/유료 플랜 확인 필요)
  - C안: **서버 개발**(가장 자유롭지만 비용/개발 범위가 커짐)

7) **추가로 꼭 필요한 정보(로고/지도/문의 전송)**
- 로고 파일 전달(파일명/형식)
- 네이버지도 임베드(iframe)용 링크 또는 “장소명/정확한 주소” 최종 확정
- 문의 전송 방식(A/B/C) 선택
